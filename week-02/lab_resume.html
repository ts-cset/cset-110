CSET-110 Lab:
Build a Resume

Today, you're going to write a resume. But this won't be like any resume you've made before, so don't bother reaching for Microsoft Word or Google Docs. Instead, we're writing it in HTML.

Like the letter example from our lecture yesterday, all resumes have a similar structure. They are built with similar parts. It's your job to take those parts and apply meaning with the proper HTML tags. However, every resume is unique to the person who writes it, so get creative and make it your own!

Instructions

  1. Open up TextEdit and create a new document.
  2. Save it as <first name>-<last name>_resume.html. For example, mine would be zach-fedor_resume.html.
  3. Open your saved file in a browser too so you can see the output as you're working. As you make changes in your file, you'll need to save the file and then reload the browser for it to update.
  4. Have fun and experiment! Use this time to explore how the browser will react to your code. Try things that you know will break so you can see how they break. You can always undo those changes in your file.
  5. When you're done, log in to the portal and upload your file.

Rubric

  • Your file needs to be a properly structured HTML document with a correct document declaration. It should correctly separate the page contents from the page metadata. You also need a title and one meta tag to define the page's character encoding. (10pts)
  • It should contain the following elements: (20pts)
    • Heading: You need multiple <h?> elements on the page in an appropriate hierarchy to organize your page's sections.
    • Paragraph: You need at least one paragraph, but use as many as you need.
    • Anchor: You need at least one external link and one internal link. They can be placed in paragraphs or as stand-alone navigation elements.
    • Lists: You need at least one of each type of list, ordered and unordered. Make sure the type you choose semantically describes the content of your list. Also, you need at least one nested list.
    • Image: You need at least one image. Make sure it has alt text.
  • You should also add at least three new tags that we haven't covered in class. Use the MDN HTML Reference to find something that would be useful. Use HTML comments in your source code to point out these new tags and tell me why you used them. (10pts)
  • And finally, your document should be computer AND human readable. That means your HTML should be valid so it will be rendered properly by the browsers. And it should also be readable as code by you and I. That means you should use whitespace (e.g. new lines, tabs, spaces) to make your code look well-structured and organized, not a hodgepodge of never-ending lines. (10pts)
